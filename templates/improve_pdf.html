AI PDF Reconstruction (v1.3.0)

How we transform "flat" PDFs into structured, accessible documents.

The Multi-Stage Process

  1. Technical Fix: We use Ghostscript to repair the PDF syntax, embed missing fonts, and normalize color spaces.
  2. AI Analysis: Using the Marker engine, we analyze the visual layout to detect headers, tables, and lists.
  3. Semantic Tagging: The reconstructed content is converted into valid HTML and then rebaked into a PDF/UA compliant container.

Requirements & Limitations

AI reconstruction is extremely resource-intensive. Therefore:

  • Local Only: This feature is currently only available when running the tool in a local Docker environment with at least 4GB of RAM.
  • Fly.io: On Fly.io, the "Improve" checkbox will automatically fall back to the technical Ghostscript fix to prevent server crashes.
